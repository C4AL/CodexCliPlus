# AGENTS.md

本仓库默认面向 Codex 协作，目标是把现有 CPA-UV、Codex 本地模式切换、插件市场与相关桥接能力收敛到单一桌面产品 `Cli Proxy API Desktop`。

## 仓库目标

- 唯一主产品：`Cli Proxy API Desktop`
- 唯一主平台：Windows
- 唯一主语言：中文
- 唯一主入口：桌面应用 + 后台服务

## 当前阶段

当前阶段优先级：

1. 冻结架构与产品边界
2. 收拢旧项目资产与历史职责
3. 建立统一桌面端、后台服务、更新中心和插件市场
4. 再逐步迁移 CPA-UV 和相关桥接逻辑

当前阶段禁止默认做法：

- 不要扩展到 Linux、macOS 或 WSL
- 不要扩展多语言 UI
- 不要引入与 CPAD/Codex 主链路无关的第三方集成
- 不要把旧仓再次拆成多个新的核心主仓

## 已确认产品边界

- 安装路径：`C:\Users\<主用户>\Cli Proxy API Desktop\`
- 运行形态：`Windows Service + 用户态 Chromium 前端`
- 开机静默启动，未登录也运行
- 主程序 `exe` 直接放安装根目录
- 所有真实数据统一放安装目录内
- 旧路径仅保留最小桥接
- 默认全新安装，不自动导入旧私密配置
- 只支持自有插件集合
- 统一更新入口
- Codex CLI 版本更新受桌面端兼容策略约束

详细条目以 [docs/已确认事项.md](docs/已确认事项.md) 为准。

## 开发协作要求

- 任何新功能设计先检查是否符合 [docs/开发计划.md](docs/开发计划.md)。
- 任何范围扩大前，先确认是否破坏“Windows-only / 中文-only / 单目录安装 / 统一更新入口”。
- 涉及旧仓迁移时，先记录来源与归属，再实施合并。
- 优先做能支撑安装、启动、切换、更新、持久化和插件管理的基础设施。
