# Cli Proxy API Desktop 已确认事项

以下条目为当前已确认的产品边界与实现方向。

## 产品边界

- 产品唯一主名：`Cli Proxy API Desktop`
- 只支持 `Windows`
- 只支持 `中文`
- 不再提供其他平台与多语言选项

## 安装与目录

- 默认安装路径：`C:\Users\<主用户>\Cli Proxy API Desktop\`
- 主程序 `exe` 直接放安装根目录
- 所有真实数据统一放安装目录内
- 允许在安装目录内建立 `data` 子目录存放 SQLite 和其他持久化数据
- 不使用 WSL 作为默认运行环境

## 运行形态

- 采用 `Windows Service + 用户态 Chromium 前端`
- 开机即启动
- 未登录也运行
- 桌面前端只负责交互与管理，不承担后台常驻职责

## Codex 接管

- 终端里 `codex` 的入口由桌面版接管
- 使用机器级 `codex` shim 转发到安装目录内的官方 Codex CLI 私有运行时
- 桌面端可切换官方模式 / CPA 模式
- 切换仅影响之后新启动的 Codex 会话
- 切换时需要明确弹窗提示用户该行为
- 不鼓励也不支持用户直接通过 `npm` 升级受控 Codex CLI

## 更新策略

- 统一更新入口
- 桌面端统一管理前端、后端、模式控制和更新流程
- Codex CLI 更新遵循官方更新源
- 但是否放行升级，由桌面端兼容策略决定
- 插件市场有独立插件更新源，但入口统一在桌面端内

## 数据与迁移

- 默认全新安装
- 不做首次导入向导
- 不自动导入旧私密配置、凭证和隐私内容
- 真实数据迁移到安装目录后，旧路径只保留最小桥接
- 如后续需要导入旧环境，放到设置页中手动触发，且默认只导入非敏感内容

## 插件市场

- 只支持自有插件集合
- 插件市场内置在桌面端
- 暂不扩展泛化第三方插件生态

## 依赖约束

- 只纳入 CPAD 与 Codex 主链路所需的必要依赖和前置
- 不主动引入无关第三方组件

## 旧项目处理方向

- 旧仓将逐步归档
- 有用资产后续并入新项目
- `Cli Proxy API Desktop` 作为后续唯一主仓库持续演进
