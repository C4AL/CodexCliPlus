## 本项目的处理原则

`Cli Proxy API Desktop` 不是要否认 `CPA-UV / CLIProxyAPI` 的来源关系，而是要在统一产品形态下继续演进。因此，本项目遵循以下原则：

1. 只要仓库内包含来自 `CLIProxyAPI`、`CPA-UV` 或其重要衍生部分的代码、资源或实质性重构内容，就继续保留其开源许可链路。
2. 本仓库采用 `MIT License`，并在主 `LICENSE` 文件中保留上游版权声明。
3. 后续如整合旧仓代码、脚本、静态资源、管理中心代码、桥接代码或插件资产，必须先确认其来源与许可证，再决定并入方式。
4. 若未来并入的某些资产具有不同许可证，必须单独标注，不得混淆为统一来源。

## 对外表述建议

对外介绍本项目时，建议统一表述为：

`Cli Proxy API Desktop` 是在 `CLIProxyAPI` 与 `CPA-UV` 现有能力基础上，面向 Windows 中文场景进行统一桌面产品化、运行时管理与插件市场整合的新主项目。
