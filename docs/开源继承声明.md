## 目的

本文件用于明确 `Cli Proxy API Desktop` 与 `CLIProxyAPI / Cli-Proxy-API-Management-Center / CPA-UV` 之间的开源继承关系，避免后续在代码迁移、重构、发布和对外说明时出现来源与许可证链路不清晰的问题。

## 当前确认的上游链路

- 官方主程序基线：`router-for-me/CLIProxyAPI`
- 官方管理中心基线：`router-for-me/Cli-Proxy-API-Management-Center`
- 当前本地继承分支：`Blackblock-inc/CPA-UV`
- 当前产品主仓：`Blackblock-inc/Cli-Proxy-API-Desktop`

说明：

- `CLIProxyAPI` 是当前主程序能力的官方基线来源
- `Cli-Proxy-API-Management-Center` 是当前管理中心能力拆解的官方来源
- `CPA-UV` 是当前覆盖层与兼容行为的直接本地来源
- `Cli Proxy API Desktop` 是新的统一产品主仓

## 本项目的处理原则

`Cli Proxy API Desktop` 不是要否认 `CLIProxyAPI / Cli-Proxy-API-Management-Center / CPA-UV` 的来源关系，而是要在统一产品形态下继续演进。因此，本项目遵循以下原则：

1. 只要仓库内包含来自 `CLIProxyAPI`、`Cli-Proxy-API-Management-Center`、`CPA-UV` 或其重要衍生部分的代码、资源或实质性重构内容，就继续保留其开源许可链路。
2. 本仓库采用 `MIT License`，并在主 `LICENSE` 文件中保留上游版权声明。
3. 官方双基线目前只是受控跟踪和同步来源，不等于已把其代码全部并入主仓。
4. 后续如整合旧仓代码、脚本、静态资源、管理中心代码、桥接代码或插件资产，必须先确认其来源与许可证，再决定并入方式。
5. 若未来并入的某些资产具有不同许可证，必须单独标注，不得混淆为统一来源。

## 对外表述建议

对外介绍本项目时，建议统一表述为：

`Cli Proxy API Desktop` 是在 `CLIProxyAPI`、`Cli-Proxy-API-Management-Center` 与 `CPA-UV` 现有能力基础上，面向 Windows 中文场景进行统一桌面产品化、运行时管理、官方基线同步与插件市场整合的新主项目。
